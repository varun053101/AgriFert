# 🌱 AgriFert — AI-Powered Fertilizer Recommendation System

AgriFert helps farmers get precise, data-driven fertilizer recommendations by combining a trained machine-learning model with real-time weather data and AI-generated soil health tips.

```
AgriFert/
├── backend/
│   ├── server/          # Node.js / Express REST API
│   └── ml_service/      # Python / Flask ML microservice
└── frontend/            # React + Vite + TypeScript SPA
```

---

## Architecture

```
Browser (React SPA)
       │  HTTP + JWT
       ▼
 Express Server  ──── MongoDB Atlas
       │  internal key
       ├──► Flask ML service  (RandomForest VotingClassifier)
       ├──► OpenWeatherMap API
       └──► Google Gemini API  (soil health tips)
```

| Service       | Tech Stack                          | Default Port |
|---------------|-------------------------------------|:---:|
| `server`      | Node 18 · Express · Mongoose        | 5000 |
| `ml_service`  | Python 3.11 · Flask · scikit-learn  | 8000 |
| `frontend`    | React 18 · Vite · TypeScript · Tailwind | 5173 |

## Quick Start

### 1 — Clone

```bash
git clone <your-repo-url>
cd AgriFert
```

### 2 — ML Service

```bash
cd backend/ml_service
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

pip install -r requirements.txt
cp .env.example .env        # edit PORT, MODEL_VERSION, ML_SERVICE_API_KEY

# Train the model (first run only)
python train_model.py

# Start the Flask server
python app.py
```

### 3 — Express Server

```bash
cd backend/server
npm install
cp .env.example .env        # fill in all required values (see .env.example)
node server.js
```

### 4 — Frontend

```bash
cd frontend
npm install
cp .env.example .env        # set VITE_API_URL=http://localhost:5000
npm run dev
```

Open **http://localhost:5173** in your browser.

## Environment Variables

### `backend/server/.env`

| Variable | Required | Description |
|---|:---:|---|
| `PORT` | ✓ | Express server port |
| `NODE_ENV` | ✓ | `development` or `production` |
| `MONGODB_URL` | ✓ | MongoDB Atlas connection string |
| `JWT_SECRET` | ✓ | 64-byte random hex for access tokens |
| `JWT_EXPIRES_IN` | ✓ | e.g. `7d` |
| `JWT_REFRESH_SECRET` | ✓ | 64-byte random hex for refresh tokens |
| `JWT_REFRESH_EXPIRES_IN` | ✓ | e.g. `30d` |
| `ML_SERVICE_URL` | ✓ | URL of the Flask service e.g. `http://localhost:8000` |
| `ML_SERVICE_API_KEY` | ✓ | Shared secret between server ↔ Flask |
| `WEATHER_API_KEY` | ✓ | OpenWeatherMap free-tier key |
| `WEATHER_API_BASE` | ✓ | `https://api.openweathermap.org/data/2.5` |
| `GEMINI_API_KEY` | ✓ | Google AI Studio key |
| `ALLOWED_ORIGINS` | ✓ | Comma-separated CORS origins |
| `ADMIN_SECRET_KEY` | ✓ | Passphrase to create first admin account |
| `LOGIN_LIMIT_MAX` | – | Max login attempts per window (default `5`) |
| `REGISTER_LIMIT_MAX` | – | Max register attempts per window (default `3`) |
| `ANALYZE_LIMIT_MAX` | – | Max analyze calls per window (default `20`) |

### `backend/ml_service/.env`

| Variable | Required | Description |
|---|:---:|---|
| `PORT` | ✓ | Flask port (default `8000`) |
| `FLASK_ENV` | – | `development` enables auto-reload |
| `ML_SERVICE_API_KEY` | ✓ | Must match value in server `.env` |
| `MODEL_VERSION` | – | Semantic version tag e.g. `1.0.0` |

### `frontend/.env`

| Variable | Required | Description |
|---|:---:|---|
| `VITE_API_URL` | ✓ | Base URL of the Express server |

## Key Features

- 🤖 **ML Prediction** — VotingClassifier ensemble (trained on NPK + soil/crop data) returns fertilizer name and confidence score
- 🌦️ **Weather Integration** — Real-time temperature, humidity, and moisture pulled from OpenWeatherMap
- 🧠 **Gemini AI Tips** — Personalised soil health tips generated by Google Gemini
- 🔐 **JWT Auth** — Access + refresh token pair; role-based admin access
- 🛡️ **Security** — Helmet, express-mongo-sanitize, rate limiting, CORS allow-list
- 📊 **Admin Dashboard** — Aggregate stats, full prediction history, user management
